# 🌞 기상위성 기반 자외선(UV) 지수 예측 모델 개발

## 1. 프로젝트 개요
- **대회명**: 2022 날씨 빅데이터 콘테스트
- **주제**: 기상위성 자료를 활용한 여름철 자외선 산출 모델 개발
- **수상**: 최우수상 (환경부장관상)
- **주관**: 기상청
- **수행 기간**: 2022.08
- **역할**: 데이터 수집 / 분석 / 모델 개발

---

## 2. 문제 정의
기상위성 데이터를 활용하여 자외선(UV) 지수를 정확하게 예측하는 모델을 구축하는 것이 목표였습니다.  
기존 방식은 단순 변수 기반 접근이 많았으며, 변수 간 상관관계 및 시간적 특성을 충분히 반영하지 못하는 한계가 있었습니다.

---

## 3. 데이터 수집 및 전처리
- R을 활용하여 ‘기상청 날씨마루’ 및 외부 데이터 수집
- 2020~2021년 6월 데이터 활용
- 자외선 지수(UV) 결측치 제거

### 주요 처리 과정
- 변수 간 상관관계 분석 수행
- 고상관 feature 그룹 파악 후 파생 변수 생성
  - `mean_5_24`, `mean_7_22` 등
- NDVI / NDWI 등 외부 변수 추가
- Rolling 기법을 활용한 시계열 특성 반영
- Grouping 기반 Feature Engineering 수행

---

## 4. Feature Engineering
- 상관성이 높은 변수 그룹 평균값 생성
- 불필요 변수 제거
  - 위도, 경도, 시간, 날짜 등
- 데이터 버전 관리 (v1 ~ v4)

### 데이터 버전별 특징
- **v1**: 주요 변수 선택
- **v2**: NDVI/NDWI 및 파생 변수 추가
- **v3**: Rolling 기법 적용
- **v4**: Grouping + Feature 정제 (최종 모델)

---

## 5. 모델링 및 실험
다양한 모델을 적용하여 성능 비교 진행

### 사용 모델
- CNN
- XGBoost
- LightGBM
- TabNet
- FFNN

### 학습 기법
- L1 / L2 Regularization
- Dropout
- Learning Rate Decay
- 5-Fold Cross Validation

---

## 6. 성능 결과

### 베이스라인 성능 (RMSE)
- CNN: 0.75  
- XGBoost: 0.66  
- TabNet: 0.65  
- LightGBM: 0.61  

### 최종 성능
- **XGBoost (v4)**  
  → **RMSE: 0.580522**

-> Feature Engineering을 통해 성능 대폭 개선

---

## 7. 핵심 개선 과정
초기에는 모델 구조 개선 중심으로 접근했으나 성능 향상이 제한적이었습니다.  
이후 데이터의 구조와 변수 간 관계를 재분석하여 Feature Engineering 중심으로 접근 방식을 변경했습니다.

- 고상관 변수 그룹화 및 평균값 생성
- 불필요 변수 제거
- 외부 데이터 반영

→ 데이터 중심 접근을 통해 모델 성능을 크게 향상시킬 수 있었습니다.
